The goal

  1. Execute automated tests on phones/tablets
  2. Siesta
  3. Automated execution/reporting
  4. Re-usable for desktop browsers

Architecture

  • Webserver with application and siesta hosted
  • Phone loads Siesta harness/index page and runs tests automatically in full screen

How it should work

  1. Start tests via webdriver CLI
  2. Tests are executed on real device
  3. Report generated to file
  4. Push test results to TestRail

How it works now

  1. Load Siesta UI manually
    • w/ UI - page view is deformed, extremely slow perf
    • w/o UI - not trivial to get results
  2. Tests auto start
  3. Simple JSON Report generated

How is it developed

  • Siesta - Touch
  • Javascript

Integrations

  • Generate JSON file based on test results
  • Push results to TestRail using Filip's trr tool
