OpenAM UI tests

PyForge tests for OpenAM in Selenium



Created by Filip Vavera <filip.vavera@profiq.com>

What is goal?

  1. To have automated UI tests for OpenAM
  2. Run them in various browsers on various platforms
  3. Run them continuously
  4. ...definitely task for Profiq!

Architecture

  • OpenAM running in Tomcat (on local system or remotely)
  • Tests in Python
  • Robot to execute these tests

How it works

  1. Deploy local OpenAM (PyForge do that)
  2. Open OpenAM in browser (Selenium)
  3. Run tests in browser

Usage

  1. Clone PyForge repo
    • git clone https://stash.forgerock.org/scm/qa/pyforge.git PyForge
  2. Configure PyForge for OpenAM
    • ./configure OpenAM
    • Set OpenAM version (12.0.1) and browser
  3. Run OpenAM UI tests
    • ./run-pybot.py --category ui OpenAM
    • ./run-pybot.py --category ui --suite policy_editor OpenAM
    • ./run-pybot.py --category ui --suite login.Registration OpenAM
    • ./run-pybot.py -c ui -s login.Login -t BlankPassword OpenAM

How is it developed

  • Developed in python (2.6.6 <= python --version > 3.0.0)
  • Using Selenium
